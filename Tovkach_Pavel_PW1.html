a.HTML5 became a standard in October 2014.
The main reasons for developing the new standard were to improve the functionality and performance of websites, address the evolving needs of web developers, and to provide better support for multimedia content such as audio and video. HTML5 also aimed to make it easier for developers to create web applications that could work across different devices and platforms.
b.The W3C HTML5 specification and the WHATWG HTML Living Standard are two different approaches to the development and maintenance of the HTML standard.
The W3C HTML5 specification is a stable, completed version of the HTML standard that was finalized in October 2014. It is a snapshot of the language at that time and is not expected to be updated further. The W3C releases incremental updates and errata, but the specification itself is considered final. On the other hand, the WHATWG HTML Living Standard is a continuously updated version of the HTML standard maintained by the Web Hypertext Application Technology Working Group (WHATWG). This version of the standard is constantly evolving and incorporates ongoing changes and updates to the language. It is designed to be a living document that reflects the current state of web development and will be continuously updated as new features and improvements are introduced. In summary, the W3C HTML5 specification is a finalized version of the HTML standard, while the WHATWG HTML Living Standard is an ever-evolving document that is continually updated to reflect the latest developments in web technologies.

d.Semantic markup refers to the use of HTML elements to convey the meaning or structure of the content on a webpage, rather than just its appearance. Semantic markup helps search engines and assistive technologies understand the content of a webpage, making it more accessible and easier to index. Presentational markup, on the other hand, is the use of HTML elements to control the visual presentation of the content, such as font styles, colors, and layouts. This approach focuses on how the content looks rather than what it actually means.
Presentational markup is considered deprecated because it separates the content from its intended meaning, making it harder for search engines to understand and index the content accurately. It also hinders accessibility for users who rely on screen readers and other assistive technologies. Additionally, presentational markup can lead to inconsistent styling across different devices and platforms, as it does not adapt well to responsive design practices. In contrast, semantic markup provides a clear structure and meaning to the content, helping both humans and machines better understand the purpose and context of the information presented. It promotes better accessibility, search engine optimization, and overall consistency in design and presentation. Overall, semantic markup is preferred over presentational markup in modern web development practices due to its focus on content structure, accessibility, and search engine optimization.
